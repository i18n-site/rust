# 优化原则

### 1. 极致性能：单次迭代 与 零拷贝

- 合并遍历：严禁不必要的重复遍历，思考如何将多轮循环、链式操作（如多次 map、filter、collect）合并为单次遍历，将时间复杂度逼近 O(N)。
- 状态机优先：处理复杂文本、流数据或协议解析时，优先设计清晰的单次循环状态机。
- 避免分配：尽量减少堆内存分配。优先使用切片 &str 或 &[u8] 代替 String 或 Vec，利用 Rust 的借用机制实现零拷贝。

### 2. 高内聚低耦合：顶层设计与模块化

- 消除逻辑重复：精准识别代码中的重复逻辑或通用模式，将其合理抽取为工具函数、独立结构体或关联方法。
- 状态机闭环：优化状态转移矩阵，确保状态定义清晰（优先使用 enum 配合 match）、无死循环、无非法状态，逻辑完全闭环。

### 3. 现代化 Rust 语法

- 拒绝陈旧风格：禁止写出 C 语言风格的 Rust 代码（如盲目使用索引遍历 for i in 0..len）。
- 现代特性：充分利用现代 Rust 特性，包括但不限于模式匹配的高级用法、优雅的迭代器链式组合、现代化的错误处理（利用 Result、? 操作符或 and_then）、适当使用 if let 或 while let 简化分支。

### 4. 可以在编译期计算的，不要放到运行时

避开硬编码,魔法数字，用常量，可在编译期计算的常量。

### 5. 不需要向上兼容，清理冗余、重复代码

进行 rust 代码审查，按以下清单逐项检查和优化：

1. 写法现代，用 rust 最新发行版的写法

- 标准库的 fs 支持 try_lock
- trait 支持返回 impl Future 的方式定义异步函数
- 复用重复的代码逻辑，可通过函数抽象、泛型参数或泛型 trait 减少冗余

2. 逻辑正确

- 确保代码无潜在 BUG
- 代码必须符合生产环境要求和行业最佳实践

3. 性能与复杂度

- 时间复杂度和空间复杂度尽量最优化
  - 尽量用流、迭代器按需生成，让空间复杂度是 O(1) 而不是 O(n)
- 对性能敏感的场景，优先考虑高效算法与数据结构。
  - 多用迭代器，少于 for，可以减少越界检查
  - 对于可以提前终止的，要避免全遍历（find、scan、take_while、try_for_each+ControlFlow），能不 collect，就避免 collect
- 尽可能消除异步的闭包捕获，手动精简捕获的变量，只捕获必要的字段，而不是整个 &self
- 大胆用 get_unchecked、unwrap_unchecked、在 100%不会出现问题的地方减少开销和越界检查

4. 架构与模块化

- 模块低耦合、高内聚。拆分过于庞大的文件为模块文件夹，拆分过于庞大的函数为几个模块文件
- 暴露的接口清晰优雅，易于使用和维护。
- 对于模块内部用的结构体，直接曝光字段，而不是写包装函数
- 兼顾性能与设计，避免过度设计

5. 代码风格

- 函数名、变量名简短清晰，禁用冗长名称。
- 避免雷同字符串，如有，请定义为常量。
- 注释用中文

6. 约束与导入

- 保持约束最小化，避免冗余。
- 使用到的模块或函数尽量在文件开头统一导入。

7. 错误处理

- 仅在 100% 确定安全时使用 unwrap。
- 禁止使用 panic，除非绝对不会触发。
- 错误在 error.rs 或独立模块中定义，避免直接使用字符串。对应依赖库的错误，用 thiserror 的 #[error(transparent)] 透明转发

8. 类型转换与安全

- 使用 as 进行数字转换时要谨慎静默失败，但如果 100% 安全可大胆使用以提升性能。
- 尽量避免数组越界检查，但仅在 100% 安全的场景下。

9. 性能优化

- 尽量避免 clone 和 Arc 等额外开销。
- 尽可能使用高性能库，例如：roaring、memchr、coarsetime、crc32fast、fastrand、rapidhash
